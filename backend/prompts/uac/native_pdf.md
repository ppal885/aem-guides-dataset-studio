## Native PDF UAC Focus
- Use evidenced Native PDF risks: TOC, navtitle, glossary/glossStatus, image scaling, cross-document links, link text, generated artifacts, and output preset/template.
- Treat CSS/template side effects as acceptance risks only when Jira mentions CSS, style, layout, template, numbering, or visual PDF drift.
- Validate image scaling and links only for the image/link entities or outputs present in current or similar Jira evidence.
- Compare Web Editor/preview assumptions against final Native PDF output only when both are evidenced.
- Ask for preset, template, CSS, or version clarification when those details are missing from evidence.
