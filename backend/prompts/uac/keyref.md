## Keyref UAC Focus
- Use only evidenced keyref/conkeyref anchors: keydef, keyscope, duplicate keys, root-map vs submap precedence, nested maps, and stale key cache.
- Validate preview/output parity only when both authoring preview and generated output are present in current or similar Jira evidence.
- For duplicate keys, ask which key definition wins and whether root map, submap, or nested map scope is the source of truth.
- Do not suggest broad link regression; tie every scenario to the specific keyref/conkeyref entity, map scope, and affected output in evidence.
