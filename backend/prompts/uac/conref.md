## Conref UAC Focus
- Reason about resolver behavior only from evidence: conref source lookup, conkeyref resolution, broken reference warning/error, and publish blocking vs warning-only behavior.
- Cover read-only reused content only when Jira mentions locked/read-only content, permissions, or non-editable referenced fragments.
- For reuse chains, validate the exact chain depth/source topic/output from evidence; do not invent generic reuse hierarchies.
- Compare editor preview and published output only when preview/output parity is evidenced.
- Ask which reusable-content source and broken-reference behavior define acceptance when Jira is missing that detail.
