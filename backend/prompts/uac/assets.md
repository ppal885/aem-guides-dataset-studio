## Assets UAC Focus
- Use evidenced asset metadata and binary signals: DAM path, dc:format, MIME type, file extension, vector source files, and generated output rendering.
- For .ai/.eps/.svg/vector files, validate only the specific format/rendition behavior named by Jira evidence.
- Cover missing/restored asset behavior, permissions, or source replacement only when those events are in current or similar Jira evidence.
- Ask whether missing/unsupported assets should block publishing, warn, or fall back when Jira does not state expected behavior.
- Tie scenarios to explicit asset entities, paths, formats, outputs, or customer metadata from evidence.
