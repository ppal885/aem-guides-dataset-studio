Resolve Keyref in Nested Keyscope
shortdesc

Fix keyref resolution failures by adding the keyscope prefix to cross-scope references in AEM Guides 4.2.

context

In AEM Guides 4.2, keyrefs defined in nested keyscopes require an explicit scope prefix in parent maps. This changed from version 4.1 behaviour where relative references resolved correctly.

steps
1.
cmdOpen the child map containing the keydef in Oxygen XML Editor
2.
cmdVerify the keyscope attribute on the map element matches the expected scope name
3.
cmdUpdate all keyrefs in the parent map: replace my-key with child.my-key
4.
cmdRegenerate output and verify key resolution in Preview panel
result

Keyref resolution works correctly across nested keyscope boundaries. Output validates without errors in AEM Guides 4.2 and later.
